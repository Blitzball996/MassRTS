## 问题诊断

### 1. **树木"看到内部"Bug** ✓ 已修复
**问题描述**：截图显示树木外表面不可见，只能看到内部面
- **根因**：`mesher.h` 的 `emit_face()` 函数使用了错误的winding order（顺时针CW），但OpenGL设置为`GL_CCW` front face，导致外表面被背面剔除
- **验证**：对+X面，原winding为 `p100 → p101 → p111`，叉乘得到 `(-1,0,0)` 指向-X（与法线相反）

**修复方案**：
- 文件：`src/sdfcraft/mesher.h` (line 96-115)
- 改动：反转所有6个cube face的顶点顺序，确保CCW winding（从外部看逆时针）
- 结果：树木外表面正确显示，内部被剔除

### 2. **地面颜色单调** ✓ 已改进
**问题描述**：地表只有草/沙/雪三种，按高度简单分配，无biome变化
- **根因**：`world.h surface_block()` 只根据海拔判断

**修复方案**：
- 文件：`src/sdfcraft/world.h` (line 388-407)
- 改动：
  - 添加biome noise（512块尺度）和温度noise（400块尺度）
  - 冷biome → 雪地斑块
  - 热+干biome → 沙漠区域
  - 高海拔 + 低温 → 雪山
  - 默认 → 草原
- 结果：更多样化的地表，biome过渡自然

### 3. **地下地质层次单调** ✓ 已改进
**问题描述**：地下只有简单的石头层，无深度变化

**修复方案**：
- 文件：`src/sdfcraft/world.h` (line 367-379)
- 改动：
  - y < 6：深层基岩石（bedrock transition）
  - y < h-8：石头主层
  - h-8 到 h-3：次土壤层（subsoil）
  - h-3 到 h：土壤层
  - h：表面块（grass/sand/snow根据biome）
- 结果：更真实的地质分层，挖掘体验更丰富

### 4. **画面灰暗无活力** ✓ 已改进
**问题描述**：整体画面偏暗，缺乏色彩活力

**修复方案**：
- 文件：`shaders/sdfcraft_chunk.frag` (line 268-276)
- 改动：
  - ambient: 0.35 → 0.50 (提升43%环境光)
  - diffuse: 0.55 → 0.70 (提升27%太阳光)
  - sky_amb: 0.10 → 0.18 (提升80%天空反射)
  - 添加暖色调阳光：`vec3(1.05, 1.00, 0.92)` (金色小时效果)
  - 添加冷色调天空反射：`vec3(0.85, 0.92, 1.10)` (蓝天反射)
- 结果：更明亮、更有活力的画面，色彩层次更丰富

## 树木系统重构

### 5. **Voxel风格可砍伐树木** ✓ 已实现

#### 5.1 粗壮多块树干
**Oak树（橡树）**：
- 2x2粗壮树干基座（高度的前50%）
- 上半部分收窄为1x1细树干
- 树干高度：5-12块（根据随机种子）

**Pine树（松树）**：
- 2x2粗壮树干基座（高度的前50%）
- 上半部分收窄为1x1
- 树干高度：8-14块

#### 5.2 树枝系统
**Oak树枝**：
- 4个主枝干（东西南北方向）
- 从trunk的3/4高度开始延伸
- 每个树枝2-4块长，末端略微上翘
- 所有树枝都由LOG块构成，可单独砍伐

**Pine树枝**：
- 锥形层状结构，无横向树枝
- 针叶林风格的密集锥形树冠

#### 5.3 物理倒塌系统
**实现**：
- 文件：`src/sdfcraft/world.h` (line 344-419)
- 函数：`check_tree_collapse(int wx, int wy, int wz, drops*)`

**算法**：
1. 当LOG或LEAVES块被破坏时触发
2. BFS洪水填充查找所有连接的树木块（LOG/LEAVES，3x3x3邻域）
3. 检查所有找到的块是否有地面支撑（向下追踪到非树木固体块）
4. 如果无支撑，移除所有悬空块并作为物品掉落
5. 玩家自动获得所有掉落物品

**特性**：
- 砍断树干底部 → 整棵树倒塌
- 砍断单个树枝 → 只有该树枝和上面的叶子倒塌
- 支持多树干树木（2x2基座需要全部砍断才倒塌）
- 防止"浮空树"bug

**触发点**：
- 文件：`src/sdfcraft/mode.h` (line 361-372)
- 集成到`do_dig()`挖掘函数中
- 破坏LOG/LEAVES块后自动检测并处理倒塌

## 技术细节

### Winding Order 修复
```cpp
// 修复前（错误的CW winding）：
case 0: n={1,0,0}; a=p100; b2=p101; c2=p111; dd=p110; break; // +X面
// 顶点顺序：100 → 101 → 111
// 叉乘：(101-100) × (111-100) = (0,0,1) × (0,1,1) = (-1,0,0) ← 指向-X，错！

// 修复后（正确的CCW winding）：
case 0: n={1,0,0}; a=p100; b=p110; c=p111; d=p101; break; // +X面
// 顶点顺序：100 → 110 → 111
// 叉乘：(110-100) × (111-100) = (0,1,0) × (0,1,1) = (1,0,0) ← 指向+X，正确！
```

### 树木物理算法复杂度
- 时间复杂度：O(N) where N = 连接的树木块数量（通常 < 500）
- 空间复杂度：O(N) for visited set
- 优化：使用std::set快速查重，3x3x3邻域确保捕获所有树枝连接

### Biome Noise Scale
- Biome noise：1/512 → 大尺度区域（数百块）
- Temperature noise：1/400 → 中尺度温度带
- 确保biome边界自然过渡，无突变
