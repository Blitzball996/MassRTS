# MassRTS

> 🌐 **语言 / Language**：[English](README.md) · **简体中文**

基于自研 OpenGL 引擎的**大规模即时战略（RTS）+ 大军团战斗模拟**。目标是在同屏渲染
**数万到数十万单位**，并配合可在运行时实时雕刻、挖掘隧道的**体积可破坏地形**。

> 状态：活跃原型阶段。GPU 构建（`MassRTS_GPU`）是主目标——单位模拟与战斗跑在 GPU
> compute shader 上。详细路线图、设计笔记与已知问题见
> [`PROJECT_PROGRESS.md`](PROJECT_PROGRESS.md)。

---

## 亮点

- **海量单位** —— GPU 驱动的移动、空间哈希与战斗（compute shader），约 19 万单位下
  仍保持可交互帧率。
- **体积可破坏地形** —— 地面是真正的有符号距离场（SDF），用分块 Marching Cubes 渲染
  （有厚度的实体，而非高度场表皮）。可在运行时挖坑、堆山、钻隧道。
- **游戏内地形雕刻** —— 升高 / 挖坑 / 平滑 / 隧道笔刷，半径与力度可调，并带贴合地表的
  笔刷光标。
- **生物群系地形** —— 草地、森林、沼泽、高山（带雪冠）、河流与水体，且影响单位移动速度
  （上坡变慢）。
- **自治 AI 指挥官** —— 据点系统：派先锋推进、扎营建 HQ、造兵营、产出增援（Phase 1）。

## 构建

需要 **CMake ≥ 3.20** 与 **C++20** 编译器。依赖（GLFW、GLAD、GLM）通过 CMake
`FetchContent` 自动拉取，无需手动配置。

```bash
cmake -B build
cmake --build build --config Release --target MassRTS_GPU
```

GPU 目标需要 **OpenGL 4.3+**（compute shader），推荐 4.6。

会生成两个可执行文件：

| 目标            | 说明                                    |
| --------------- | --------------------------------------- |
| `MassRTS_GPU`   | 主构建 —— GPU 加速的模拟与战斗。        |
| `MassRTS`       | 原始 CPU 构建（稳定参考实现）。         |

从构建输出目录运行，以便找到 `shaders/` 和 `assets/` 文件夹：

```bash
cd build/Release
./MassRTS_GPU.exe
```

### 自动截图模式

```bash
./MassRTS_GPU.exe --shots
```

预热后摄像机绕预设角度环绕，每个角度输出一张 PNG，然后退出——便于快速从多个视角检查地形。

## 地形架构（简述）

地形以有符号距离场（SDF）存储在稀疏的 32³ 体素分块中。只有表面穿过的 chunk 才分配内存
并网格化（Marching Cubes）；纯空气 / 纯实心 chunk 保持稀疏。传统高度场保留作为单位 /
摄像机 / 战斗的廉价查询面。

SDF 经过**梯度归一化**（`距离 ≈ (y − h) / √(1 + |∇h|²)`），让网格在陡坡上保持平滑，
而不是碎裂成尖刺。挖掘是确定性事件 `{中心, 半径, 操作, 力度}`，每个客户端应用相同编辑会
得到完全一致的状态（适配 lockstep 联机）。挖掘被钳制在 slab 底面之上，因此可以挖很深，
但永远不会把地图打穿。
